# Telegram Бот для проверки подписки

Этот проект — Telegram-бот с WebApp для проверки подписки пользователя на канал и выдачи секретного контента только подписчикам.

## Возможности
- Проверка, подписан ли пользователь на указанный канал
- Веб-приложение для удобной проверки
- Выдача эксклюзивного контента только подписчикам
- Сообщения и интерфейс полностью на русском языке

## Быстрый старт

### 1. Клонируйте репозиторий
```bash
git clone <ссылка_на_репозиторий>
cd sub-checker-tg
```

### 2. Установите зависимости
```bash
npm install
```

### 3. Настройте переменные окружения
Создайте файл `.env` в корне проекта и добавьте:
```
BOT_TOKEN=ваш_токен_бота
WEBAPP_URL=https://ваш_домен_или_ngrok
CHANNEL_ID=@ваш_канал
SECRET_LINK=https://example.com/secret
PORT=3000
```
- **BOT_TOKEN** — токен вашего Telegram-бота
- **WEBAPP_URL** — URL, по которому доступно ваше веб-приложение (используйте ngrok или свой сервер)
- **CHANNEL_ID** — username или ID вашего канала (например, @my_channel)
- **SECRET_LINK** — ссылка на секретный контент
- **PORT** — порт для запуска сервера (по умолчанию 3000)

### 4. Запустите сервер
```bash
node index.js
```

### 5. Настройте WebApp в BotFather
- В BotFather используйте команду `/setdomain` и укажите ваш WEBAPP_URL
- В `/setcommands` добавьте команду `/start`

### 6. Использование
- Пользователь пишет боту `/start`
- Бот отправляет кнопку для проверки подписки через WebApp
- Веб-приложение проверяет подписку и выдает ссылку на контент только подписчикам

## Структура проекта
- `index.js` — основной сервер и логика бота
- `public/` — статические файлы веб-приложения (HTML, CSS, JS)

## FAQ
**Q:** Как узнать свой CHANNEL_ID?
**A:** Используйте @username вашего канала или его числовой ID (например, -1001234567890).

**Q:** Как развернуть на хостинге?
**A:** Используйте любой Node.js-хостинг (Heroku, Render, VPS). Не забудьте прописать переменные окружения.

**Q:** Как сделать ссылку на секретный контент уникальной?
**A:** Можно генерировать уникальные ссылки для каждого пользователя в обработчике подписки.

## Советы
- Для тестирования используйте [ngrok](https://ngrok.com/) для проброса порта наружу.
- Не публикуйте свой BOT_TOKEN и SECRET_LINK в открытом доступе.
- Для продакшена используйте HTTPS.
